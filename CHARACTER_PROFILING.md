# 群聊赛博相面（性格凝练）系统设计

## 1. 核心目标
将高频、冗长且碎片化的群聊消息，转化为结构化、高密度、持久化的用户“性格画像”（Cyber Profile）。
解决群聊机器人长期记忆爆炸（Context 溢出）的问题，使得机器人在任何时候都能用极低的 Token 成本“认识”每一位群友。

## 2. 凝练机制（双轨增量压缩模型）
我们采用 **“单次话题切片（Snapshot） + 全局画像融合（Merge）”** 的双轨处理策略。

### 第一轨：单次切片（Analyze & Summarize）
当触发群聊总结时（如艾特机器人发送“总结”），大模型（DeepSeek）会对**未被总结过**的近期聊天记录进行阅读。
大模型的任务不仅是提炼聊天内容（Topic Summary），更重要的是进行**微观行为抓取**：
* **提取动作**：针对这次聊天，提取出活跃发言者的表现。
* **脱水输出**：将发言者的行为压缩成一句核心特征描述（`trait`），例如：“在讨论股票大跌时表现出异常的乐观”或“面对技术争论时喜欢用表情包敷衍”。
* **规则过滤**：通过 System Prompt 强制要求模型忽略无意义的唤醒指令（如“@机器人 总结”），只关注有社交价值的发言。

### 第二轨：全局融合（Profile Merge）
这是实现**持久化长期记忆**的核心步骤。每次提取到单次切片（`trait`）后，系统并不会简单地将其 append 到一个无限变长的列表里，而是触发一次 **LLM 记忆压缩**。
* **读取旧档**：从本地硬盘（`profiles/{group_id}_{user_id}.json`）加载该群友的 `historical_summary`（历史综合画像）。
* **冲突与演进处理**：将“旧画像”与“新表现（`trait`）”同时喂给大模型，让大模型扮演**心理分析师**。
  * 如果新表现印证了旧画像（如一直是吃货），大模型会加深该标签的权重。
  * 如果新表现颠覆了旧画像，大模型会更新认知（如：“虽然平时是个技术宅，但今天暴露出其实是个二次元”）。
* **尺寸限制**：通过 Prompt 强控融合后的 `historical_summary` 不超过 **150 字**，并提取出 3 个以内的核心 `traits` 标签（如 `["技术宅", "暴躁", "深夜党"]`）。

## 3. 存储与流转结构

```json
// 存储结构示例 (profiles/733746606_123456.json)
{
  "user_id": "123456",
  "nickname": "数字泔水搬运师",
  "traits": [
    "热衷技术探索",
    "发图狂魔"
  ],
  "historical_summary": "平时极少参与闲聊，但一涉及代码报错、开源项目等技术话题就会长篇大论。最近在讨论A股时展现出一定的悲观情绪。偶尔喜欢用魔性表情包刷屏。"
}
```

### 数据流转流程
1. **收集**：群聊消息进入 SQLite 数据库。
2. **提取**：`character_parser.py` 中的 `analyze_and_summarize` 提取单次表现。
3. **融合**：调用 `merge_character_traits` 融合旧档案和新表现。
4. **覆写**：将新压缩的不到 200 字节的 JSON 覆写回本地文件。
5. **展示**：通过 `render_card.js` 截取最新发言时的昵称和单次特质，渲染至总结图片卡片；而其全局的 `historical_summary` 潜藏在冰山之下，随时可供后续调用。

## 4. 后续应用场景拓展（设想）
有了这份凝练好的高密度 JSON，未来机器人可以实现：
* **@某某 查水表**：一键生成该群友的“赛博体检报告卡片”。
* **千人千面的回复**：当机器人主动回复某群友时，偷偷将该群友的 `traits` 放入 system_prompt：“请注意，向你提问的这个人性格是【暴躁老哥】，请用简洁甚至互怼的语气回复他”。
* **群生态报告**：每周生成一张“群聊生态分布图”，直观展示本周“杠精”、“摸鱼王”、“大明白”的人员变动。
